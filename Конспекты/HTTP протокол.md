**Клиент-серверное взаимодействие** - архитектура взаимодействия в сети, при которой клиент отправляет запрос на сервер в ожидании того, что сервер обработает этот запрос и либо вернет какие-то данные, либо изменит данные в месте хранения.

## HTTP

**HTTP** - протокол клиент-серверного взаимодействия. С помощью него передаются веб-страницы, файлы данных (JSON, текст, бинарники). Также протокол хранит информацию о состоянии соединения в рамках текущего запроса.

#### Запрос

**Запрос** - специальное сообщение для сервера, которое определяет, что хотел клиент при его отправке

```
POST /api/users/123 HTTP/1.1       ← Метод, адрес (URI) и версия протокола
Host: example.com                  ← Заголовки
User-Agent: Mozilla/5.0
Accept: application/json
Authorization: Bearer abc123

{"key": 123}                       ← Тело запроса (для POST/PUT, для GET тело запроса не используется)
```

**HTTP-методы**:
- **GET** - получение данных, без тела запроса: страница, данные пользователя
- **POST** - отправка данных на сервер: данные пользователя для внесения его в базу
- **PUT** - полное обновление ресурса на сервере (замена): перезапись данных пользователя
- **PATCH** - частичное обновление ресурса на сервере: обновление некоторых полей данных пользователя
- **DELETE** - полное удаление ресурса: удаление данных о пользователе

Из всех запросов только GET не меняет данные. Этот метод позволяет для одних и тех же запросов всегда получать один и тот же ответ (например, запрос данных конкретного пользователя) - это называется **идемпотентность**

**Адрес (URI)** - адрес (или же эндпоинт) по которому производится запрос. Он может включать в себя параметры: **path** и **query**. В примере выше 123 - path-параметр идентификатора пользователя (`/api/users/{userID}`). Query-параметры являются дополнительными и указываются после `?`, разделяются `&`: `/api/users/{userID}?offset=20&limit=2`

**Заголовки запроса** - некоторые метаданные, техническая информация для сервера. Например, формат тела запроса, авторизация и т.д.

Финально запрос (кроме **GET**) подкрепляется **телом запроса** - данными, с которыми сервер будет работать, к примеру, данные о пользователе, которые необходимо внести в базу данных.

#### Ответ

После обработки запроса сервером следует ответ

```
HTTP/1.1 200 OK                     ← Версия протокола, код и статус ответа
Content-Type: application/json      ← Заголовки ответа
Content-Length: 1234
Date: Mon, 23 Oct 2023 12:00:00 GMT
Server: nginx/1.18.0

{                                   ← Тело ответа
  "id": 123,
  "name": "John Doe",
  "email": "john@example.com"
}
```

**Коды ответа**:
- `2xx` - успешные ответы
- `3xx` - перенаправление запроса
- `4xx` - ошибки клиента
- `5xx` - ошибки сервера

[Все статус-коды](https://developer.mozilla.org/ru/docs/Web/HTTP/Reference/Status)
