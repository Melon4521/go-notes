# Пакет io

## Интерфейсы `io`

Пакет `io` содержит в себе полезные интерфейсы:

- `io.Reader` - типы, читающие байты из ресурса и записывающие их в `p` методом `Read(p []byte) (int, error)` (возвращает число прочитанных байтов и ошибку)
- `io.Writer` - типы, записывающие байты из `p` в ресурс методом `Write(p []byte) (int, error)` (возвращает число записанных байтов и ошибку)
- `io.Closer` - типы, закрывающие ресурс методом `Close() error`
- `io.ReadWriter` - объединяет интерфейсы `io.Reader` и `io.Writer`
- `io.ReadCloser`, `io.WriteCloser`, `io.ReadWriteCloser` - объединяют интерфейсы `io.Reader`+`io.Closer` / `io.Writer` + `io.Closer` / все вместе



# Пакет os. Работа с файлами и директориями

## Работа с файлами

Основные операции с файлами:
- Открытие файла
- Чтение файла
- Запись в файл
- Создание файла
- Проверка существования файла

Взаимодействие с файлами происходит через пакет `os`. Файл в нем представляется в виде следующей структуры:

```go
type File struct {
	*file // внутреннее представление файла
}
```

> [!important]
> Операции с файлами (методы структуры File) являются **конкурентно безопасными**

#### Открытие файла

Функции `os.Open(filename)` и `os.OpenFile(filename, flag, perm)` позволяют получить доступ к файлу. Первая функция имеет ограничение: она может открыть файл **только** для чтения (под капотом эта функция вызывает `os.OpenFile` с флагом на чтение). 

Функции возвращают указатель на структуру найденного файла и ошибку. Если файл не найден, то возвращается ошибка типа `*fs.PathError`.

После завершения работы с файлом его необходимо закрыть через метод `file.Close()` (он может возвращать ошибку, которую нужно будет обработать). Закрывать файл необходимо для освобождения ресурсов:

```go
func main() {
	file, err := os.Open("input.txt")
	if err != nil {
		return err
	}
	
	// гарантируем закрытие и обрабатываем ошибку
	defer func() {
		if err := file.Close(); err != nil {
			log.Fatal(err)
		}
	}()
}
```

Функция `os.OpenFile(filename, flag, perm)` позволяет открыть файл не только для чтения. Для этого необходимо указать [специфичные флаги](https://github.com/golang/go/blob/go1.25.6/src/os/file.go#L79) вторым параметром (можно объединять их через побитовое ИЛИ `|`). Основные флаги:
- `os.O_RDONLY` - только чтение
- `os.O_WRONLY` - только запись (перезапись)
- `os.RDWR` - чтение и запись
- `O_APPEND` - добавление в конец файла при записи
- `O_CREATE` - создать файл, если не существует

Третьим аргументом передаются права доступа к файлу (как в Linux)

```go
// чтение+запись, создать если нету
file, err := os.OpenFile("notes.txt", os.O_RDWR|os.O_CREATE, 0644)

// обработка err и закрытие...
```

#### Чтение файла

Для чтения файла целиком можно воспользоваться функцией `os.ReadFile(filename)`, она вернет слайс байтов (содержимое) и ошибку:

```go
data, err := os.ReadFile("data.txt")

if err != nil {
	log.Fatal(err)
}

fmt.Print(string(data)) // преобразуем слайс байтов в строку
```

Прочитать файл **порциями** можно с помощью метода `file.Read()`. Он возвращает количество прочитанных байтов и ошибку (реализует интерфейс `io.Reader`). В него нужно передать слайс-буфер байтов заданного размера `make([]byte, n)`, этот размер показывает сколько байт за раз мы хотим прочитать из файла. Чтобы вычитать весь файл, нам необходимо вызывать `Read()` итеративно, пока не закончится содержимое для чтения. Если в какой-то момент в файле остается ровно 0 байт для чтения, то возвращается ошибка `io.EOF`:

```go
file, err := os.Open("data.txt")
if err != nil { log.Fatal(err) }
defer file.Close()

buffer := make([]byte, 1024) // читаем 1 Кб за раз

for {
	n, err := file.Read(buffer)
	
	if n > 0 {
		someWorkWithBuffer(buffer[:n]) // имитируем работу с порцией
		// buffer[:n] - берем только ту часть, которую прочитали
	}
	
	if err == io.EOF {
        break // файл закончился, выходим из цикла
    }
    
    if err != nil {
        fmt.Printf("Ошибка при чтении: %v\n", err)
        break
    }
}
```

#### Эффективный способ чтения

Более производительным способом чтения файлов (на самом деле любых ресурсов, которые имплементируют интерфейс `io.Reader`) является функция `bufio.NewReader(r io.Reader)` пакета `bufio`. Она не обращается к жесткому диску за каждым байтом, а считывает сразу кусок (по умолчанию 4 Кб) в память, после чего быстро отдает нам байты. 

Функция возвращает нам указатель на структуру `bufio.Reader`. Её основные методы:
- `Read(p []byte)` - стандартное чтение в буфер (но с кэшированием)
- `ReadString(delim byte)` - сплитит файл по `delim` и читает одну часть (если передать `\n`, то будет чтение по строкам)
- `Peek(n int)` - возвращает слайс с `n` следующими взятыми байтами, при этом курсор чтения (на каком байте в файле сейчас находимся) не сдвигается.

```go
f, err := os.Open("file.txt")
if err != nil { log.Fatal(err) }

reader := bufio.NewReader(f)

for {
	line, err := reader.ReadString('\n')
	
	if err == io.EOF {
		break // все прочитано
	}
	
	if err != nil {
		log.Fatal(err)
	}
	
	fmt.Println(line)
}
```

Еще есть `bufio.NewScanner(r io.Reader)`, которая возвращает структуру `bufio.Scanner`, позволяющая читать файлы последовательно. Пример чтения по строкам:

```go
scanner := bufio.NewScanner(file)

// цикл идет, пока есть что читать
for scanner.Scan() {
    line := scanner.Text() // возвращает string без \n в конце
    // или scanner.Bytes(), если нужны []byte
    fmt.Println(line)
}
```

#### Запись в файл

Открыть файл (создать, если его нет) и записать в него слайс байтов можно с помощью `os.WriteFile(filename, data, perm)`:

```go
data := []byte("Full content of the file")
err := os.WriteFile("config.json", data, 0644)
```

Также можно записать в файл напрямую через структуру:
- `file.Write([]byte)` - записывает слайс байтов (реализует интерфейс `io.Writer`)
- `file.WriteString(string)` - удобная обертка, чтобы не делать `[]byte(s)` вручную

```go
f, err := os.Create("test.txt")
defer f.Close()

n, err := f.WriteString("Hello Go!") // возвращает количество байтов и ошибку
```

#### Создание файла

Создать файл можно с помощью функции `os.Create(filename)` (или если он уже существует, очистить и открыть):

```go
file, err := os.Create("file.txt")
if err != nil { log.Fatal(err) }

// работаем с файлом...
```

#### Проверка существования файла

Чтобы проверить, существует ли определенный файл, можно использовать `os.Stat(filename)` (функция для получения метаданных файла) и проверить наличие ошибки `fs.ErrNotExist`:

```go
func fileExists(filename string) (bool, error) {
	_, err := os.Stat(filename)
	if errors.Is(err, fs.ErrNotExist) {
		return false, nil
	}
	
	if err != nil {
		return false, err
	}
	
	return true, nil
}
```

## Работа с директориями

Через пакет `os` также можно взаимодействовать с директориями.

#### Создание

- `os.Mkdir(name, perm)` - создает одну директорию
- `os.MkdirAll(path, perm)` - создает все директории по пути, которых еще не существует

```go
err := os.MkdirAll("data/logs", 0755)
if err != nil { log.Fatal(err) }
```

#### Просмотр содержимого

Функция `os.ReadDir(path)` возвращает слайс сущностей директории, файлы или другие директории, они удовлетворяют интерфейсу `fs.DirEntry` с методами `Name() string`, `IsDir() bool`, `Type() FileMode` (возвращает тот самый код доступа `perm`), `Info() (FileInfo, error)`

#### Рекурсивный обход

Пройтись по содержимому директории рекурсивно можно через:
`filepath.WalkDir(startPath string, func (path string, d fs.DirEntry, err error) error)`. Второй аргумент - функция-обработчик для каждой вложенной сущности.

## Аргументы запуска

При запуске скрипта на Go можно передавать аргументы. Все они доступны в слайсе `os.Args`. Первым элементом всегда является путь до скрипта:

```go
fmt.Println("Аргументы:")

for i, arg := range os.Args {
	fmt.Printf("%d: %s\n", i+1, arg)
}
```

```
go run main go 1 2 3

Аргументы:
1. /tmp/go-build123456/b001/main
2. 1
3. 2
4. 3
```

#### Пакет `flag`

Удобнее работать с аргументами запуска можно через пакет `flag`. Он предоставляет удобный способ определения именованных параметров (флагов) с типизацией, значениями по умолчанию и автоматической генерацией справки. Он также автоматически обрабатывает различные форматы передачи флагов (`-name=value`, `-name value`, `--name=value`, `--name value`). Булевы флаги можно указывать без значения (`-verbose` то же самое, что и -`verbose=true`).

Сначала мы объявляем необходимые для работы флаги через: `flag.T(flag, default, usage)`. После парсим через `flag.Parse()`. В итоге в объявленные переменные флагов присваиваются указатели на спаршеные значения (или значения по умолчанию):

```go
var (
	name    = flag.String("name", "Мир", "имя для приветствия")
	count   = flag.Int("count", 1, "количество повторений")
	verbose = flag.Bool("verbose", false, "подробный вывод")
)

flag.Parse()

if *verbose {
	fmt.Printf("Выполняется приветствие с параметрами: name=%s, count=%d\n", *name, *count)
}

n := *count
for range n {
	fmt.Printf("Привет, %s!\n", *name)
}
```

```bash
go run main.go -name="Алиса" -count=3 -verbose
```

```
Выполняется приветствие с параметрами: name=Алиса, count=3
Привет, Алиса!
Привет, Алиса!
Привет, Алиса!
```

Чтобы получить слайс неименованных аргументов (множественные аргументы какого-то флага), используем `flag.Args()`.
